Directive: esp-input-group

<esp-input-group ...>
Synopsis
<esp-input-group ng-model="NAME" fields="FIELD,FIELD,..." filters="FILTER,FILTER,..." inputClasses="CLASS,CLASS,..." labels="LABEL,LABEL..." labelClasses="CLASS,CLASS,..." types="TYPE,TYPE,...">
Description

The esp-input-group generates a set of appropriate input element based on the types of the input fields for a record as determined from the database schema. The esp-input-group directive can generate "input", "textarea", "checkbox", "radio" and "select" form elements and utilizes the esp-input directive to do this.

Attributes may be passed to esp-input via comma separated attribute strings. For example: to override the default input labels, the "labels" attribute may contain a comma separated list of labels for each field.

The esp-input-group directive supports the named attributes below. In addition, it will pass unknown attributes through to the input element.

If a label is not specified, the name of the relevant field is used.

Attributes
fields: String Comma separated list of field names. If unspecified, defaults to the order of fields specified in the database record schema.
filters: String Comma separated list of Angular filters to apply to the value before display. This is useful to format dates or numbers.
inputClasses: String Comma separated list of CSS styles to use for each input.
labels: String Comma separated list of labels text for each input field.
labelClasses: String Comma separated list of CSS styles to use for the input labels.
ng-model: String Name of the model field to input.
types: String Comma separated list of input types to render, (checkbox, text, textarea, date, email, tel, url, search, color, number, range).
Example
<esp-input-group ng-model="event.id"></esp-input-group>
<esp-input-group type="text" ng-model="event.date" filter="date:'yyyy-MM-dd HH:mm:ss'"></esp-input-group>
<esp-input-group type="textarea" ng-model="event.message" rows="6"></esp-input-group>
<esp-input-group ng-model="event.title"></esp-input-group>
